# 製品仕様詳細リファレンス

個人スキルレビュー用の仕様要約。計算の正本は [`docs/spec.md`](../../docs/spec.md)、要件の正本は [`docs/REQUIREMENTS.md`](../../docs/REQUIREMENTS.md)。

## 1. プロダクト定義

| 項目 | 内容 |
|------|------|
| 目的 | 育成完了時に「取得可能スキルをすべて取る」ための**理論必要 SP**を算出する |
| 想定ユーザー | 対人レギュやガチ育成を想定し、必要 SP を事前見積もりしたいプレイヤー |
| 形態 | ブラウザ静的アプリ（インストール不要） |
| 公開版 | v0.1.7（2026-07-14） |
| 非公式性 | 非商用・非公式。権利は各権利者に帰属 |

### スコープ境界（やらないこと）

スキルレビューでは「何を切ったか」も重要な成果物である。

| スコープ外 | 理由 |
|------------|------|
| トレセン軒以外のシナリオ | 当面約4ヶ月想定で固定 |
| 全サポカのイベント網羅 | 優先37種のみ |
| 継承固有の親名・固有名 | 汎用行のみで最小化 |
| スタミナ等の最終パラメータ | SP 習得コストのみ |
| Electron / 必須バックエンド | 静的公開で足りる |
| プリセットデッキ保存 | 需要確定まで保留 |

## 2. 利用者向け機能一覧

### 2.1 入力

| 機能 | 仕様 |
|------|------|
| 育成ウマ娘選択 | 検索付きセレクト。初期値あり（ナリタトップロード系） |
| サポカ6枠 | 重複不可。検索・SSR・タイプ・**イベント対応のみ**（デフォルト ON） |
| 切れ者 | 全スキルに追加 10% OFF |
| 継承固有 | ON/OFF、個数 2–6、ヒント Lv 1–5、baseSp 既定 200 |
| イベント | デッキ内の優先サポカイベントのみ表示。`auto` / `single` |
| シナリオリンク | ラジオ1択・未選択なし。編成連動で白/金切替 |
| シニア RMJ | ラーメン3択（選択金が変わる） |
| シナリオ自動 | ガチ想定で常時計上（盛況トグルなし） |

### 2.2 出力・操作

| 機能 | 仕様 |
|------|------|
| 合計必要 SP | 結果パネル + 画面下部固定バー。差分ハイライトあり |
| スキル一覧 | 含めるチェック・名前・発動条件タグ・ヒントLv・必要SP・由来 |
| スキル数 | `N/M`（ON / 全件。継承固有含む） |
| 「含める」除外 | 行単位で合計から外す（対人レギュ想定） |
| レギュ絞込 | バ場／距離／作戦。**適用ボタンで確定**（draft/committed） |
| 説明書 | ヘッダー「使い方」→ `<dialog>`（FAQ・制限事項） |

### 2.3 レギュ絞込（v0.1.7）

ゲーム本家の「モーダル決定」に寄せたセマンティクス（UI はセレクト＋適用）。

| 状態 | 意味 |
|------|------|
| draft | 3セレクトの表示値（変更だけでは合計は変わらない） |
| committed | 実際の互換判定・増分除外に使う確定値 |

| 操作 | 挙動 |
|------|------|
| セレクト変更のみ | 未適用表示（枠色＋文言）。合計不変 |
| 「適用」クリック | 非互換を全置換除外。手動「含める」もリセット |
| 編成・イベント等でスキル集合変化 | **新規 skillId のみ**確定済み絞込を増分適用。手動状態は維持 |
| 一覧から消えた skillId | `excludedSkillIds` から prune（再登場は新規扱い） |

## 3. 計算仕様（要約）

### 3.1 ヒント Lv 解決

由来ごとにヒントを集め、**最大値**を採用（`hintResolve.js`）。

| 由来 | Lv |
|------|-----|
| サポカ訓練ヒント | 5 |
| 育成ウマ娘所持スキル | 3（他が高ければ上書き） |
| イベント / シナリオ | JSON 定義 |

### 3.2 SP コスト式

```text
cost = floor(baseSp * (1 - hintDiscount - (切れ者 ? 0.10 : 0)))
```

| ヒント Lv | 割引 |
|-----------|------|
| 0 | 0% |
| 1 | 10% |
| 2 | 20% |
| 3 | 30% |
| 4 | 35% |
| 5 | 40% |

### 3.3 購入チェーン

| ケース | 扱い |
|--------|------|
| 金 + 白 | 白未取得のまま金を取る場合 `cost(白)+cost(金)`。一覧は金行に合算表示 |
| ○ + ◎ | ヒントは○準拠。○のみなら◎行に合算。金がある場合は下位全段合算 |
| ×（`group_rate < 0`） | 購入チェーン外 |
| 固有スキル本体 | SP 購入リスト外（一覧に出ない） |
| 覚醒進化（金→金） | 進化前の金 ID のみ計上 |

検証例は `docs/spec.md` および `npm test` に固定。

### 3.4 継承固有

```text
calcSkillCost(baseSp, hintLv, 切れ者) * 個数
```

親名・固有名は扱わない（スコープ最小化）。

## 4. データモデル（アプリが読むもの）

| ファイル | 生成 | 内容の要約 |
|----------|------|------------|
| `data/skills.json` | extract | 約 2103 件。baseSp、上下位リンク、`activation` |
| `data/supports.json` | extract | 約 543 件。ヒント skillId、レア・タイプ |
| `data/characters.json` | extract | 約 261 件。`skillsByAwakening` |
| `data/events.json` | U-tools 抽出正本 | 優先37・102イベント + preserve |
| `data/scenarios/toresenken.json` | 手メンテ | リンク白/金・RMJ・終了スキル |
| `data/meta.json` | extract | 抽出元パス・件数・日時 |

### 発動条件タグ（`activation`）

- mdb `skill_data` の precondition / condition から抽出
- 軸: バ場（芝/ダート）、距離（短/マイル/中/長）、作戦（逃げ〜追込）
- OR 分岐・チェーン OR マージを `skillActivation.js` で扱う

## 5. ランタイム構成

```text
index.html
  └─ app.js
        ├─ fetch JSON
        ├─ UI 状態（編成・イベント選択・除外 Set・committed 絞込）
        └─ buildSkillPlan(aggregate.js)
              ├─ ヒント収集（サポカ / キャラ / イベント / シナリオ）
              ├─ resolveHintLevels
              ├─ filterDisplaySkills / calcAcquisitionCost（goldLower）
              └─ 合計 + 行一覧
```

| モジュール | 責務 |
|------------|------|
| `spCost.js` | 割引テーブルと単体コスト |
| `hintResolve.js` | max(hintLv) |
| `goldLower.js` | 白金・○◎チェーンと表示フィルタ |
| `skillActivation.js` | 発動条件・絞込・増分除外の pure 関数 |
| `scenarioLink.js` | リンク白/金の編成連動 |
| `aggregate.js` | 全由来の集計入口 |
| `app.js` | UI・再計算・除外状態 |

## 6. UX / UI 上の設計判断（レビュー観点）

| 判断 | 内容 |
|------|------|
| ガチ想定固定 | 盛況段階トグルを廃止し、説明コストと誤入力を削減 |
| イベント対応のみデフォルト ON | 優先37に寄せ、未対応サポカでの「イベントが無い」混乱を減らす |
| 適用ボタン | 本家の決定操作に寄せ、select 即時適用による手動除外破壊を避ける |
| 増分除外 | デッキ変更で新規スキルだけ絞込を効かせ、手動調整を維持 |
| 説明書モーダル | 計算ルールをアプリ内に持ち、外部 wiki 依存を減らす |
| 合計バー固定 | スクロール中も SP を見失わない |

## 7. 制限・免責（ユーザー向け告知の骨子）

- シナリオはトレセン軒のみ  
- イベントは優先37サポカのみ  
- ゲーム更新後はデータが古くなる場合がある  
- 非公式・非商用  

告知文の現行版は `README.md` およびアプリ内説明書を参照。
