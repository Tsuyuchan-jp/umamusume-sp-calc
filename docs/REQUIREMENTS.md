## 確定ルール（変更禁止に近い）

### ヒントLv

| 由来 | ヒントLv |
|------|----------|
| サポカトレヒント | 3–5（既定5・オプション） |
| 育成ウマ娘所持スキル | 3（他由来がより高ければその値） |
| イベント / シナリオ | JSON 定義値 |
| 同一スキルが複数由来 | `max(levels)` |

用語: [GLOSSARY.md](./GLOSSARY.md)

### 金スキル + 白スキル（実機検証済み）

```
cost = floor(baseSp * (1 - hintDiscount - (切れ者 ? 0.10 : 0)))
```

ヒント割引:

| Lv | 割引 |
|----|------|
| 0 | 0% |
| 1 | 10% |
| 2 | 20% |
| 3 | 30% |
| 4 | 35% |
| 5 | 40% |

白未取得のまま金を取る場合:

```
cost(白, whiteHintLv) + cost(金, goldHintLv)
```

検証例:

| 操作 | SP |
|------|-----|
| 強者の証 Lv2 のみ | 170 |
| さらなる高みへ Lv0 を直取り（白未取得） | 306 |
| 白取得後に金 | 136 |

mdb は白・金それぞれ別 `baseSp` を持つ（合算済みではない）。

### ○ + ◎（ガチ想定・軽量版）

- ヒントは **○ に付く**（◎専用ヒントは無い。◎ ID でも白帯で共有）
- ○のみヒント → **◎まで取得扱い**で一覧・合算（金へは自動繰り上げしない）
- 金が計画に入る → 下位チェーン全段（○+◎+金）を合算し、下位行は非表示
- 「〇も取らない」は表示行の除外で表現
- **×（`group_rate < 0`）** は購入チェーン外。extract / `relink_skills.mjs` でリンクしない

### その他ルール

- **固有スキル本体**: SP 購入対象外（extract は `available_skill_set` のみ → 一覧に出ない）
- **覚醒進化（金→金）**: 購入リストは進化前の金 ID。進化後 ID は計上しない（mdb・実機確認済み）
- **育成ウマ娘覚醒レベル**: 最大想定で所持スキルを全合算
- **切れ者**: UI トグル（全スキルに追加 10%OFF）
- **除外**: 一覧からスキルを外して合計から落とせる（対人レギュ等）。結果の **バ場／距離／作戦** 絞込で非互換スキルを一括除外も可（絞込は「適用」で確定。確定時に手動除外はリセット。編成変更時は確定済み絞込を新規スキルに増分適用）
- **結果スキル件数**: 結果パネル見出し右に **スキル数 N/M**（ON＝合計に含まれるスキル本数、全件＝一覧のスキル本数。通常行は1行＝1本。継承固有は個数分を本数に加算）。実機確認済み（v0.1.10）
- **含めるスキルコピー**: 見出し右ボタン。含める ON の表示行スキル名をカンマ区切り1行でクリップボードへ（継承固有は対象外。外部ツール向けエクスポート）。実機確認済み
- **継承固有オプション**: 汎用行のみ。個数 2–6、baseSp 既定 200、ヒント一律 1–5。親名・固有名は出さない（スコープ最小化）
- **トレヒントLvオプション**: 編成サポカのトレヒントを 3 / 4 / 5 で一括指定（既定5）。イベント・シナリオ由来は変更しない
- **シナリオ**: 当面 **トレセン軒のみ**（約4ヶ月想定）。参考: [umasim ramen_memo.md](https://github.com/mee1080/umasim/blob/main/data/ramen_memo.md)
- **イベント**: 優先37サポカ分。**U-tools + mdb 抽出**が正本（`npm run extract:events` → `apply:events`）。例外2件は `events.preserve.json`。全サポカ一括はスコープ外。トレヒントは mdb 自動。
- **サポカ UI 絞込**: 「イベント対応のみ」はデフォルト ON（`prioritySupportIds` で id 照合。名前マッチは使わない）。選択中カードはその枠に残す。
- **結果スキル絞込**: バ場／距離／作戦（未選択＝全 ON）。`skills.json` の `activation` で互換判定。セレクト変更は下書きのみ、「適用」で確定（手動「含める」もリセット）。編成変更時は確定済み絞込を新規 skillId に増分適用
